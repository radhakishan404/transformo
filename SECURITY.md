# Security Policy

## Supported Versions

Security fixes are applied to the latest release branch.

## Reporting a Vulnerability

Please do **not** open public GitHub issues for security vulnerabilities.

Instead:

1. Contact maintainers privately.
2. Include reproduction steps and impact.
3. If possible, include a proof-of-concept and suggested mitigation.

We will acknowledge valid reports and aim to provide a fix or mitigation as quickly as possible.
